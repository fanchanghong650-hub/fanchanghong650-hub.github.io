# Probability Theory Blog Writing Guide (概率论博客写作规范)

## 触发条件

当用户要求对《Introduction to Probability》(Blitzstein & Hwang, 2nd ed.) 的某一章（或任何数学教材的某一章）写博客风格的总结文章时，按照本规范执行。

## 核心原则：2/8 法则 + 概念理解优先

博客文章的目标是用 80/20 法则提炼一章中最重要的 20% 概念，以**深层概念理解**而非公式记忆为导向。读者应该能从文章中理解"为什么这些概念重要"以及"它们之间的关联是什么"。

## 文件位置与命名

- 博客文章存放路径：`C:\blog\source\_posts\`
- 命名格式：`probability-ch{章节号}-{英文关键词}.md`
- 参考：`probability-ch2-conditional-probability.md`, `probability-ch3-random-variables.md`, `probability-ch4-expectation.md`

## 前置检查清单

写文章之前，必须完成以下准备：

1. **教材内容确认**：确定本章的标题、主要小节和不适用的小节
2. **Stat 110 教案查阅**：用 WebSearch 和 WebFetch 查找 Harvard Stat 110 对应的 Strategic Practice (SP) 和 Homework (HW)，获取实际的习题列表和解答
3. **习题筛选**：从 SP/HW 中精选 7-8 道最能代表本章核心概念的习题，标注每道题的出处（如 SP4-2.3）
4. **渲染安全检查**：确认所有 LaTeX 在 Hexo + MathJax 环境下的兼容性

## 文章结构模板

每篇文章必须包含以下 8 个部分（可根据章节特点调整顺序）：

### 1. 标题与 Frontmatter

```yaml
---
title: 概率论第X章「{中文标题}」— {副标题}
date: YYYY-MM-DD HH:00:00
categories:
  - 数学
  - 概率论
tags:
  - {核心概念标签}
  - Stat110
  - Blitzstein
mathjax: true
---
```

Tags 应覆盖章节的核心概念（如：期望, 方差, 协方差, 条件期望, 全期望定律）

### 2. 开篇引言

- 解释本章在全书中的位置和重要性（"为什么这一章值得你花时间"）
- 与其他章节的关系（前承什么、后启什么）
- 用一句核心洞察概括全章

### 3. 前置知识

用项目符号列出（3-5 条），说明需要的前置章节和具体知识。

### 4. 核心概念讲解（主体，3-5 节）

每个核心概念的讲解必须包含：
- **定义**：严格但简洁的数学定义
- **直觉**：用非数学语言解释"这意味着什么"
- **为什么重要**：这个概念解决了什么问题
- **与其他概念的关系**：它在全章体系中的位置
- **具体例子**：至少一个能体现概念本质的例子

对于离散分布，必须讲"分布的故事"——每个分布对应的数据生成过程，而不是孤立地给出 PMF。

对于定理，必须区分"需要什么条件"和"不需要什么条件"（如期望的线性不需要独立性）。

### 5. 概念全景图

用文本图 (ASCII art) 或层次描述展示本章核心概念之间的关系。

### 6. 精选习题 (7-8 道)

每道题必须包含：
- 题号 + 来源标注（如 **题 1（SP4-2.3 | 鞋带问题）**）
- 题目陈述
- **▼ 答案与解析**（用加粗标题，不用 `<details>` 标签）
- 完整的解答过程
- **洞察**（这道题教会我们什么）
- **训练点**（这道题训练的具体技能）

习题应按主题分组（3 组），每组 2-4 题。至少 4 道题应直接来自 Stat 110 SP/HW。

### 7. 本章核心脉络（回顾）

- 分层次总结全章的核心概念（2-3 个层次）
- "如果你只从这一章带走 N 样东西"——最重要的 3 个概念/工具/思维模式

### 8. 参考文献

必须包含：
1. 教材：Blitzstein & Hwang, Introduction to Probability (2nd ed.)
2. Stat 110 课程录像（标注对应的 Lecture 编号，可在 YouTube 搜索）
3. Stat 110 Strategic Practice + Homework（标注具体的 SP/HW 编号）
4. 其他参考教材（可选）

## LaTeX 渲染安全规则（极其重要）

Hexo 的 Markdown 渲染器 (hexo-renderer-marked) 在 MathJax 之前处理，以下模式会导致渲染失败：

| 禁止 | 替代方案 |
|------|----------|
| `\underbrace{}_{}` | `\underset{}{}` |
| `\overbrace{}^{}` | `\overset{}{}` |
| `<details>` HTML 标签 | 用加粗文字分隔（如 **▼ 答案与解析**） |
| `\%` 在 `$$` 数学模式中 | 在文字中使用，或替换为 `%` |

**必须**在每篇文章写完后，用 `grep` 检查是否有以上禁止模式。

## 写作风格指南

- **语言**：中文，学术但可读——像一个优秀的老师在讲解
- **公式**：关键公式用 `$$ $$` 独立展示，行内引用用 `$ $`
- **强调**：用 `**粗体**` 突出核心概念的命名和关键洞察
- **注释**：在概念之间加入反向链接（如"回顾第二章，……"、"这将在第五章……"）
- **直觉优先**：永远先讲"为什么"再讲"是什么"——在给出正式定义之前，先建立直觉
- **禁止**：不写"我们可以看到"、"值得注意的是"等空洞过渡语；不写多行注释；不写超过 3 行的段落

## 验证清单

文章完成后，检查：

1. [ ] 所有 `$$` 块是否正确闭合（奇数个 `$$` 会导致整个页面渲染失败）
2. [ ] 用 `grep` 检查无 `underbrace`、`overbrace`、`<details>`、数学模式中的 `\%`
3. [ ] 每道习题都有 **▼ 答案与解析** 部分（不是用 `<details>` 标签）
4. [ ] 参考文献包含教材、Stat 110 录像、SP/HW 三个来源
5. [ ] Frontmatter 包含 `mathjax: true`
6. [ ] Tags 覆盖了本章关键概念
7. [ ] 开头有一段解释"为什么这一章重要"的引言
8. [ ] 习题中至少 4 道与 Stat 110 SP/HW 中的题目对应
