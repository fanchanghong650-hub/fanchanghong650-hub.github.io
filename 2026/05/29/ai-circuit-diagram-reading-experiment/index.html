AI 阅读电路图的能力边界——一次系统化的工具链实验

发表于2026-05-29|更新于2026-05-29|电子工程AI工具
|浏览量:

问题的起点

让 AI 分析电路问题有一个致命瓶颈：AI 能计算，但看不懂电路图。

这就好比一个数学家被要求解题，但题目写在了一张他看不见的照片里。他可以解方程、写矩阵、算行列式——但前提是他知道电路长什么样。

本文记录了我在 2026 年 5 月 29 日进行的一次系统化实验：试图搭一个工具链，让 Claude Code 能够从图片中提取电路拓扑结构，再结合电路分析方法求解。以下是完整的方法、发现和结论。

实验设置

测试材料：五道交流电路习题的微信拍照图片，包括正弦稳态分析、RC 选频网络、共轭匹配等题型。

目标：从图片中提取电路的完整拓扑结构（元件类型 + 连接关系），然后自动求解。

基础设施：

  • Python 3.12 (安装在 D:\claude\pdf\)
  • Tesseract OCR 5.4.0 (用于文字识别)
  • Poppler 25.07 (pdftotext/pdftoppm for PDF)
  • PyTorch 2.12 + YOLOv5 (用于元件检测)
  • OpenCV + NetworkX (图像处理和拓扑图构建)

工具链路 1：纯 OCR 方案（Tesseract）

做了什么

使用 Tesseract OCR 的中文语言包（chi_sim）直接对图片进行文字识别。

结果

能力 效果
识别中文题目文字 ⭐⭐⭐⭐ 可用（~80% 准确率）
识别数学符号 ⭐⭐ 差（∠ 变成 Z，√ 变成 W）
识别下标/上标 ⭐ 很差（V1 变成 Vl，Z_L 变成 Zr）
识别电路拓扑 ❌ 完全不行

示例 OCR 输出：

  • V1 → Vl（下标 1 被识别为字母 l）
  • V2 → Va（下标 2 被识别为字母 a）
  • 10V → 10Y（V 被识别为 Y）

根本问题

OCR 只能识别文字。电路图是图形——元件符号、导线、节点——这些对 OCR 来说是不可见的。Tesseract 完全没有电路拓扑的概念。

工具链路 2：PDF 直接渲染（PyMuPDF + Poppler）

做了什么

如果电路图是 PDF 格式（而非照片），可以用 PyMuPDF 直接提取文本，或用 Poppler 的 pdftoppm 将页面渲染为图片。

安装的工具

工具 位置 用途
Python 3.12 + PyMuPDF D:\claude\pdf\ PDF 文本提取 + 页面渲染
Poppler (pdftoppm, pdftotext 等) D:\claude\pdf reader\poppler\ PDF → 图片/文本转换

结果

场景 效果
文字型 PDF（如概率论教材） ⭐⭐⭐⭐⭐ 完美
扫描版 PDF（图片页） ⭐⭐⭐ 能提取图片，但需要后续 OCR
印刷电路图 ❌ PDF 本身不包含拓扑信息

PyMuPDF 能直接从 PDF 中提取嵌入的文本，这对文字型 PDF（大多数现代电子书）效果极好——例如我们用它成功读取了概率论教材（Blitzstein & Hwang）和电路理论教材（Vlach）的全文。但对于以图片形式嵌入的电路图，这只是把问题从”PDF 中的图片”变成了”图片”——还是没有解决拓扑提取问题。

工具链路 3：电路专用图像解释器（CircuitSchematicImageInterpreter）

做了什么

克隆并测试了剑桥大学的开源项目 CircuitSchematicImageInterpreter（C. R. Kelly, CK598@cam.ac.uk）。

工作原理

  1. 二值化 + 骨架化：将电路图转化为黑白线条图
  2. Hough 变换：检测水平和垂直导线
  3. 导线间隙分析：识别元件位置（导线之间的断点 = 元件）
  4. 交叉点检测：找到节点（导线交汇处）
  5. 图构建：生成 NetworkX 图，构建生成树、余树
  6. 网表生成：输出 SPICE 仿真网表

实际测试数据（习题 1）

精确到像素的坐标数据：

元件 质心 (x,y) 方向 连接
C1 (328, 104) 水平 上轨中段
C2 (324, 151) 水平 中轨中段
C3 (252, 218) 竖直 左柱中段
C4 (411, 151) 竖直 中柱上段（仅 6px 长）
C5 (411, 216) 竖直 中柱下段
C6 (484, 216) 竖直 右中柱
C7 (540, 218) 竖直 右柱

重构的拓扑图（ASCII）

效果评估

能力 效果 说明
导线位置检测 ⭐⭐⭐⭐⭐ 精确到像素
元件定位 ⭐⭐⭐⭐ 能识别导线间隙
节点检测 ⭐⭐⭐⭐ 交叉点基本正确
拓扑图构建 ⭐⭐⭐⭐ NetworkX 图 + 生成树
SPICE 网表生成 ⭐⭐⭐ 依赖 OCR 识别元件类型
元件类型识别 ⭐ 最大瓶颈——需要自定义 Tesseract 模型

关键限制

  1. 元件类型 OCR 不可用：项目需要一个自定义训练过的 Tesseract 模型（model_project_final.traineddata），但这个文件不在公开的 GitHub 仓库中
  2. 对拍照图片敏感：微信拍照的图片（有畸变、阴影、非均匀光照）比扫描的原理图效果差
  3. 只支持二端子元件：对于晶体管等多端子器件的处理不完善

工具链路 4：YOLOv5 元件分类（Electrical_Schema_Recognition）

做了什么

使用 Electrical_Schema_Recognition 的预训练 YOLOv5 模型（modelv1.pt）进行元件类型分类。

模型信息

  • 架构：YOLOv5
  • 训练数据：150 张手绘电路图
  • 类别：capacitor, inductor, voltage source, resistor, diode, lamp（6 类）
  • 训练精度：~90% mAP（在手绘电路上）

实际测试结果

习题 1 检测（7 个元件）：

问题：

现象 原因
圆形电压表被识别为 lamp 训练数据中 lamp 是唯一的圆形符号
印刷电阻被识别为 inductor 印刷体的锯齿/矩形电阻与手绘线圈差异大
整体置信度偏低（0.25-0.51） 训练域（手绘）与测试域（印刷）不匹配
很多元件未被识别 6 个类别太少，不包含电压表/电流表/地线等

效果评估

能力 效果
元件位置检测 ⭐⭐⭐⭐
元件类型分类（手绘图） ⭐⭐⭐⭐
元件类型分类（印刷图） ⭐ 很差
类别覆盖度 ⭐ 只有 6 类

工具链路 5：其他探索过的项目

项目 状态 原因
CirAI809 ❌ 未使用 18 类印刷电路数据集，但只含图片不含模型权重
Circuitry.ai ⚠️ 未测试 YOLOv8 + LLaMA3，需要 GPU

综合方案得分低的原因：元件类型识别是瓶颈。拓扑有了，但不知道元件是 R 还是 L 还是 C，就无法计算。

根本难题

导线检测和文字 OCR是相对容易的计算机视觉问题。但元件类型识别是一个细粒度分类问题，再加上电路元件符号因标准（IEEE vs IEC）、绘制风格（手绘 vs 印刷）、尺度、旋转而变化多端——目前没有针对印刷电路教材训练的开源模型。

可能的解决方案

方案 A：人机协作（当前可行度 ⭐⭐⭐⭐⭐）

  • 优点：100% 可靠的拓扑信息，AI 的计算能力得到充分利用
  • 缺点：需要人工描述，效率受限于人的速度
  • 适用场景：精确求解、验证答案、学习辅导

这是我们目前采用的方案。

方案 B：专用模型微调（当前可行度 ⭐⭐⭐⭐）

在印刷电路数据集上微调 YOLOv8：

  • 使用 CirAI809 的数据集（601 张图，18 类）
  • 或自行标注一批教材电路图
  • 预期可将印刷电路元件分类准确率提升到 90%+

方案 C：多模态 LLM（当前可行度 ⭐⭐⭐）

使用 GPT-4o / GPT-4V 等视觉语言模型直接”看”电路图：

  • 优点：理论上一步到位，不需要搭工具链
  • 缺点：
    • 需要 API 调用（费用 + 网络）
    • 拓扑幻觉（串联/并联混淆）仍然存在
    • 对复杂电路（桥式、非平面）的准确率不足
  • 论文证据：Auto-SPICE（2024.11）测试 GPT-4o 从电路图生成网表，器件级 F1=1.0，但功能块级 F1=0.31

方案 D：端到端电路图理解系统（当前可行度 ⭐⭐）

综合 CircuitSchematicImageInterpreter（拓扑）+ 微调的 YOLOv8（元件分类）+ 训练好的 OCR（标注文字）→ 完整网表 → SPICE 仿真：

  • 优点：全自动
  • 缺点：开发和训练成本高，各模块的集成调试工作量大

结论

  1. 导线和元件的空间位置可以从图片中精确提取（像素级精度）。剑桥大学的 CircuitSchematicImageInterpreter 是这方面最成熟的开源方案。

  2. 文字题目可以通过 Tesseract OCR 提取，中文支持良好，但数学符号和下标容易出错。

  3. 元件类型识别是当前最大的缺口。没有针对印刷电路教材训练的开源模型。YOLOv5 的预训练模型只覆盖手绘电路的 6 类元件。

  4. 最实用的当前方案是人机协作：拓扑提取自动化 + 元件类型人工标注 + AI 计算。这样既利用了 CV 的精确空间定位能力，又避免了分类模型的不准确。

  5. 多模态 LLM（GPT-4o 等）代表了未来的方向，但当前的拓扑推理准确率还达不到工程可靠性的要求。需要等待针对电路图的专用微调模型。

  6. 如果 DeepSeek 未来推出多模态能力，配合电路图专用微调数据集（如 CirAI809），可能会是第一个真正可用的中文电路图 AI 分析方案。
